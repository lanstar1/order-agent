# Order Processing System - 종합 사용자 요구사항 정리

## 기본 현황 및 목표

### 현재 업무 프로세스
- 네이버 스마트스토어에서 주문상품 엑셀(스마트스토어_선택주문발주발송관리_YYYYMMDD_HHMM.xlsx) 다운로드
- ERP 양식(주문처리_마스터파일.xlsx)에 수동으로 매칭/입력
- 택배 운송장 엑셀 파일(파일변환예제문서.xlsx) 수동으로 작성
- 택배 운송장 수집 후 스마트스토어에 송장번호 수동으로 입력

### 목표
전체 프로세스 자동화: 스마트스토어 주문 엑셀 업로드 → ERP 매칭 엑셀 + 택배업로드 엑셀 자동 생성 → 택배 송장번호 수집 후 스마트스토어에 자동 입력

---

## 1. 주문처리 방식 (주문 수집 및 매칭)

### 1.1 기본 주문 그룹화 규칙
- **C열 기준 그룹화**: 스마트스토어 파일의 C열 값이 같으면 하나의 주문건으로 인식
- **상품주문번호 vs 주문번호**:
  - 상품주문번호가 다르더라도 주문번호가 동일하면 같은 고객의 주문
  - 데이터 수집 시 **주문번호 기준**으로 수집

### 1.2 ERP 상품 매칭 로직

#### 1.2.1 상품번호 직접 매칭 (A열)
- 스마트스토어 A열의 상품번호를 ERP 품목코드에 직접 매칭
- 매칭 데이터는 json 파일 형태로 서버 시작 시 자동 로드
- 일반적인 상품은 이 방식으로 처리

#### 1.2.2 모델명 기반 매칭 (E열, F열)
- **E열 제품명 확인**:
  - row11, row12 등에서 E열의 제품이 동일한 경우, 별도의 모델명 시트를 참조하여 다르게 매칭
  - 모델명(LS, LSP, LSN, ZOT)에 맞는 품목코드를 입력하여 ERP 양식에 반영

- **F열 모델명 직접 입력**:
  - F열에 모델명(LS, LSP, LSN, ZOT) 입력되어 있는 경우
  - A열 상품번호와 매칭되지 않으므로 별도의 시트(모델명-품목코드 매칭)를 통해 매칭
  - 매칭 데이터: 스마트스토어목록.xlsx + 품목코드모델명매칭.xlsx를 참고하여 자동 생성

### 1.3 미매칭 건 처리
- 50% 이상의 매칭률을 목표로 함
- 미매칭 건은 json 파일에 별도 기록하여 사용자가 확인/수동 매칭 가능하도록 함

## 2. ERP 판매입력 (ERP 양식 작성)

### 2.1 필수 입력 필드 (자동)
- **상품번호 → 품목코드 자동 매칭** (위의 매칭 로직 참조)
- **수량**: 주문 수량 자동 입력
- **단가**: 수량 × 공급가 계산
- **공급가**: 금액 정보에서 자동 추출
- **거래처코드**: 모든 주문에 동일 값 입력 (설정값)
- **담당자**: 모든 주문에 동일 값 입력 (설정값)
- **일자**: 한국표준시(KST)로 당일 날짜 자동 입력
- **출하창고**: 모든 주문에 동일 값 입력 - **창고 코드로 입력 (값: 30)**
- **공급가액**: 정산예정금액으로 입력

### 2.2 제거/변경 사항
- **순번 열**: 삭제 (불필요)
- **창고 입력 방식**: 창고명이 아닌 **창고 코드(30)**로 입력

---

## 3. 택배 송장 합묶음 (발송처리)

### 3.1 송장 그룹화 규칙
- **C열 기준 그룹화**: C열 값이 같으면 **송장 1개만 출력**
- **원칙**: 복수의 상품이 같은 주문에 포함되면 송장은 1개만 생성
- **출력 기준**: 상품주문번호가 아닌 **주문번호로 송장 묶음 출력**

---

## 4. 택배업로드 엑셀 양식

### 4.1 기본 규칙
- 파일변환예제문서.xlsx 양식에 맞게 자동 입력
- **I열(품목명)**: 품명이 아닌 **모델명(LS, LSP, LSN, ZOT)과 각 모델별 주문수량만 입력**
  - 예: "LS(2), LSP(1)" 형태 (품명은 너무 길어지므로 제외)
- **B열**: 불필요하므로 제거 (기존 양식대로 유지, B열 제거)

### 4.2 배송비 입력 규칙 (S열 기반)
- **스마트스토어 파일의 S열(배송비 유형)** 참조:
  - S열 = "착불" → **택배업로드 I열에 020 입력**
  - S열 = 그 외 → **택배업로드 I열에 030 입력**
- **T열(배송비) 참조**:
  - 묶음상품 처리: C열이 같은 경우(주문코드 동일) 배송비는 **하나만 입력** (복수 상품이어도 배송비 중복 X)

### 4.3 상품 정보
- 배송용 상품주문번호 기준으로 입력 (주문번호 아님)
- 같은 주문의 복수 상품이라도 각각의 상품주문번호로 행 생성 (단, 송장 번호는 동일)

---

## 5. 택배업로드 결과 수집

### 5.1 결과 파일 처리
- 택배사에서 반환한 결과 파일에서 **B열 제거**
- 기존 양식대로 유지

### 5.2 송장번호 수집
- **주문번호 기준 그룹화**: 상품주문번호가 달라도 주문번호가 동일하면 같은 고객
- **송장 취합**: 주문번호 기준으로 수집하여 스마트스토어에 입력할 준비

### 5.3 발주 상태 확인
- "발주확인" 안 된 주문도 수집 대상에 포함되어야 함

## 6. 웹 UI/UX 및 운영 방식

### 6.1 플랫폼 및 개발 환경
- **저장소**: https://github.com/lanstar1/order-agent.git
- **아키텍처**: FastAPI 백엔드 + 바닐라 JS SPA 프론트엔드
- **배포**: Render (Standard 유료 플랜)
- **서버 환경**: 환경변수로 외부 API 키 관리 (재배포 시에도 유지)

### 6.2 대시보드
- **스마트스토어 페이지**: 별도 탭/섹션으로 생성
- **웹 기반 처리**: 엑셀 다운로드/업로드 방식이 아닌 웹에서 직접 처리

### 6.3 주요 버튼 기능
- **주문 수집 버튼 (1클릭)**:
  - 네이버 API를 통해 주문 수집
  - 엑셀 다운로드 없이 직접 버튼 클릭으로 처리
  - 스마트스토어 설정(API 키) 필수

- **송장 입력 버튼 (1클릭)**:
  - 택배사 결과 파일 처리
  - 송장번호를 스마트스토어에 자동 입력
  - 별도의 파일 형식 변환 없이 웹에서 처리

### 6.4 기획안/문서
- **형식**: Word 문서 (Markdown이 아닌 보기 좋은 UI/UX 개선)
- **목적**: 다른 세션에서도 이해할 수 있도록 상세 기획

## 7. 주문 추가옵션 확인

### 7.1 옵션 데이터 수집
- 주문 시 추가된 옵션(색상, 사이즈, 기타 커스터마이징) 확인
- ERP 입력 시 옵션 정보 포함 여부 검토

### 7.2 송장 입력 시 옵션 표기
- 품목명 대신 모델명으로 표기하되, 주요 옵션 정보 포함 검토
- 너무 길어지지 않도록 간결하게 표기

---

## 8. 송장과 품명/모델명 처리

### 8.1 기본 원칙
- **송장에는 품명이 아닌 모델명 입력**
- 이유: 품명이 입력되면 송장 라벨이 너무 길어짐
- 모델명 + 수량으로 충분한 정보 제공

### 8.2 모델명 종류
- LS, LSP, LSN, ZOT

### 8.3 복합 상품 표기
- 같은 주문에 여러 모델의 상품이 있는 경우:
  - 예: "LS(2), LSP(1)"
  - 각 모델별로 수량 표기

---

## 9. 기술 요구사항

### 9.1 데이터 저장
- **임시 데이터**: PostgreSQL 불필요 (API 연동, 1회성 처리)
- **영구 데이터**:
  - 상품 매칭 데이터: JSON 파일 형태로 서버 시작 시 자동 로드
  - 미매칭 건: 별도 JSON 파일로 기록

### 9.2 자동화 프로세스
- **서버 시작 시 자동 로드**:
  - 품목코드-상품번호 매칭 데이터
  - 모델명-품목코드 매칭 데이터
  - 미매칭 건 목록

### 9.3 API 연동
- **네이버 커머스 API**: 주문 수집 및 송장 입력
- **IP 등록**: API 키 설정 필수 (IP 화이트리스트 설정)
- **환경변수 관리**: Render 환경변수에 저장

## 10. 알려진 이슈 및 해결사항

### 10.1 IP 관련
- Render Standard 플랜의 고정 IP 설정 필요 여부 검토 (배포 시 IP 변경되는 문제)
- API 키 설정 후 재배포 시에도 유지 필요 (환경변수 사용)

### 10.2 데이터 수집
- "과거 주문 선택" 방식보다는 현재 신규 주문만 수집하는 방식 필요
- 발주확인 안 된 주문도 수집 대상에 포함

### 10.3 성능
- 매칭률 50% 이상 목표
- 대량 주문 처리 시 성능 최적화 필요

## 11. 참고 파일

### 필수 참고 파일
1. **주문처리_마스터파일.xlsx**: ERP 양식 표준
2. **스마트스토어_선택주문발주발송관리_YYYYMMDD_HHMM.xlsx**: 스마트스토어 주문 데이터
3. **파일변환예제문서.xlsx**: 택배업로드 양식 표준
4. **스마트스토어목록.xlsx**: 스마트스토어 상품 목록
5. **품목코드모델명매칭.xlsx**: 모델명과 품목코드 매칭 정보
6. **naver api.txt**: 네이버 커머스 API 정보

### 생성될 매칭 파일
- **product_code_mapping.json**: 상품번호 ↔ 품목코드 매칭 (자동 생성)
- **model_code_mapping.json**: 모델명 ↔ 품목코드 매칭 (자동 생성)
- **unmatched_products.json**: 미매칭 상품 목록 (수동 검토/수정용)
