# 스마트스토어 주문 자동화 - 비즈니스 요구사항

이 스킬은 스마트스토어 주문 자동화 시스템의 **비즈니스 로직과 상세 요구사항**을 담고 있다.
코드 수정 시 반드시 이 요구사항을 참조하여 비즈니스 규칙이 올바르게 반영되었는지 확인해야 한다.

## 참조 문서

상세 요구사항은 references 폴더에 있다:

- `references/전체요구사항.md` — 11개 섹션으로 구성된 종합 요구사항 (업무 프로세스, ERP 양식, 택배 양식, UI/UX 등)
- `references/사용자요청_인용문.md` — 사용자의 정확한 원문 인용 (비즈니스 규칙의 근거)

코드 수정 전에 해당 영역의 요구사항을 반드시 읽고 확인할 것.

## 핵심 비즈니스 규칙 요약

### 1. 주문 그룹화
- **주문번호(C열) 동일 = 하나의 주문건**
- 상품주문번호가 달라도 주문번호가 같으면 같은 고객의 주문
- 데이터 수집 시 주문번호 기준으로 그룹화

### 2. 택배 송장 합묶음
- 주문번호 동일 시 **송장 1개만 생성** (복수 상품이어도 1개)
- 발송처리도 상품주문번호가 아닌 **주문번호 기준으로 묶음 처리**

### 3. 송장 품목명 = 모델명 + 수량
- 품명(상품명)이 아닌 **모델명(LS, LSP, LSN, ZOT) + 수량**만 입력
- 복합 상품: `LS(2), LSP(1)` 형태
- 이유: 품명이 들어가면 송장 라벨이 너무 길어짐

### 4. ERP 판매입력 양식
- 순번 열 삭제
- 창고: 코드로 입력 (값: `30`)
- 공급가액: **정산예정금액**으로 입력
- 일자: KST 당일 날짜
- 거래처코드, 담당자: 설정값 동일 적용

### 5. 택배업로드 양식
- B열: 불필요 (제거)
- I열 품목명: 모델명 + 수량 (품명 X)
- 배송비 유형: S열 "착불" → `020`, 그 외 → `030`
- 묶음상품(주문번호 동일): 배송비 **1건만** 입력

### 6. 추가옵션 확인
- 주문 수집 시 옵션 정보(색상, 사이즈 등) 확인/표시 기능 필요
- ERP 입력 시 옵션 정보 포함 여부 검토

### 7. 전체 워크플로우
```
[1클릭 주문수집] → [ERP 판매입력 엑셀 다운로드] → [택배업로드 엑셀 다운로드]
                                                          ↓
[택배사에 업로드] → [결과 엑셀 수신] → [송장번호 업로드] → [1클릭 발송처리]
```
